핵심 시나리오의 흐름. 로컬 우선·무인증 전제(진입 즉시 사용).

## 1. 첫 진입 (빈 상태 → import 유도)

```mermaid
flowchart TD
  A["앱 실행"] --> B{"데이터 있음?"}
  B -- 아니오 --> C["빈 대시보드 + '내역 가져오기' 유도"]
  C --> D["가져오기 플로우 (플로우 2)"]
  B -- 예 --> E["대시보드: 이번 달 합산 요약"]
```

## 2. 가져오기 (Import) — 제품의 심장

```mermaid
flowchart TD
  A["가져오기 진입"] --> B["파일 업로드 (CSV·엑셀, ≤10MB)"]
  B --> C["기관 자동 인식 + 인코딩 감지(CP949→UTF-8)"]
  C --> D{"컬럼 매핑 완전?"}
  D -- 아니오 --> E["수동 매핑 요청"] --> F
  D -- 예 --> F["정규화 + 중복·이체 자동 판별"]
  F --> G["미리보기: 신규 / 중복 / 오류 / 이체(제외)"]
  G --> H{"사용자 확정?"}
  H -- 취소 --> A
  H -- 확정 --> I["저장 + 출처·멤버 기록"] --> J["거래 내역으로 이동"]
```

## 3. 거래 분류·태깅 (카테고리 + 공동/개인)

```mermaid
flowchart TD
  A["거래 내역"] --> B["거래 선택 → 편집"]
  B --> C["카테고리 지정(수동 또는 자동규칙)"]
  C --> D["공동 / 개인(멤버) 태깅"]
  D --> E["저장 → 대시보드 합산·분담 반영"]
```

## 4. 이체 — 이중집계 방지 (수동 보정)

```mermaid
flowchart TD
  A["거래 내역"] --> B["이체 관리 열기"]
  B --> C{"보정 방식"}
  C -- 짝 맞추기 --> D["출금 ↔ 입금/카드 연결"]
  C -- 이체 규칙 --> E["적요 키워드 → 항상 이체"]
  C -- 일괄 --> F["여러 건 선택 → 이체로 표시"]
  D --> G["지출 집계에서 제외 (한 번만)"]
  E --> G
  F --> G
```

## 5. 카드 할부 — 보기 전환

```mermaid
flowchart TD
  A["거래 내역"] --> B["할부 지출 기준 토글"]
  B -- 구매 시점 --> C["산 달에 전액 집계"]
  B -- 월 청구 --> D["이번 달 청구분만 집계(스케줄 파생)"]
  C --> E["대시보드·예산에 관점 반영"]
  D --> E
```

## 6. 계좌·카드 등록

```mermaid
flowchart TD
  A["계좌·카드"] --> B{"추가 대상"}
  B -- 계좌 --> C["은행·계좌번호·소유(멤버/공동) 입력"]
  B -- 카드 --> D["발급사·소유 + 결제계좌 연결·결제일"]
  C --> E["레지스트리 등록 → 이체 자동판별에 활용"]
  D --> E
```

## 7. 예산 설정·점검

```mermaid
flowchart TD
  A["예산"] --> B["카테고리별 월 예산 설정"]
  B --> C["실적 대비 소진율 표시(초과=빨강)"]
  C --> D["대시보드에서 한눈에 확인"]
```

## 8. 백업 (Phase 2)

```mermaid
flowchart TD
  A["설정 → 백업"] --> B["인증(서버 사용 시점)"]
  B --> C["전체 데이터 압축 → S3 스냅샷 업로드"]
  C --> D["복원: 다운로드 → 로컬 반영"]
```
