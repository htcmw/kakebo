# ADR-0012: 백엔드 스택 (Phase 2 · 백업/복원)

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted (Phase 2) |
| 날짜 | 2026-07-08 |
| 단계 | Construction |

> 확정(2026-07-08): 방향 채택. 세부(인증 수단·암호화)는 Phase 2 착수 시 확정.

## 맥락 (Context)

로컬이 정본이므로(ADR-0007) MVP엔 서버가 없다. 서버는 **Phase 2**에서 **백업/복원**(및 향후 2인 공유)만 담당한다. 인증은 이 단계에서만 필요(ADR-0008). "S3 하나에 전체를 압축 저장" 수준의 최소 서버가 목표.

## 결정

| 영역 | 선택 |
| :-- | :-- |
| 저장 | **S3-호환 오브젝트 스토리지** — 전체 데이터셋을 압축한 단일 아카이브(타임스탬프 버저닝). **무료/오픈 우선**: Cloudflare R2(무료 10GB·egress $0) 기본, Backblaze B2 대안, 자가호스팅은 **Garage/SeaweedFS**(오픈소스). *MinIO 커뮤니티는 2026-02 아카이브되어 제외.* |
| 업로드 방식 | **프리사인 URL 직접 업로드** 우선(서버 최소) |
| 컴퓨트 | 서버리스(Cloudflare Workers / AWS Lambda 등) — 상시 서버 없음 |
| 인증 | Phase 2에서 확정(오픈·무료 우선) |
| 클라이언트 도구 | restic/Kopia/rclone 등 오픈소스 백업 도구가 S3 API를 1급 지원 |
| 동기화(향후) | 2인 실시간 공유 필요 시 오픈소스 sync engine(ElectricSQL/Yjs) 검토 |

## 근거

- 백업은 스냅샷이면 충분 → 상시 DB·서버 불필요, 비용·복잡도 최소(ADR-0007).
- 프리사인 URL로 클라이언트가 S3에 직접 올리면 서버 표면적이 거의 0.
- 서버리스는 사용량 기반이라 사이드 프로젝트 비용에 적합.

## 결과 (Consequences)

- (+) 운영 부담 최소, 필요할 때만 과금.
- (−) 스냅샷 백업은 레코드 단위 병합이 안 됨 → 2인 실시간 공유는 별도 동기화 설계 필요(향후).
- (→) 백업 아카이브 스키마·버전 호환·**암호화**를 아키텍처에서 확정.
- Phase 1(로컬 MVP) 완료 후 착수. 확정 시 Accepted로.
