# ADR-0001: 금융 자동연동 미사용, 수동 export→import 채택

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted |
| 날짜 | 2026-06-18 |
| 단계 | Inception |

## 맥락 (Context)

가계부는 은행·카드 거래내역을 어떻게 수집할지 정해야 한다. 한국에서 자동연동 경로는 사실상 셋이다.

- **오픈뱅킹 직접 연계**: 금융결제원 이용기관 등록, 사업자·보안점검·기능적합성심사 필요 → 개인/2인 프로젝트에 비현실적.
- **마이데이터(본인신용정보관리업)**: 금융위 허가 + 자본금 등 요건. 허가 없이 영위 시 위법 → 불가능.
- **서드파티 어그리게이터(예: CODEF)**: 유료 계약 + 사용자 금융 자격증명/민감정보 취급 → 보안·법적 책임이 MVP 초기부터 과중.

## 결정 (Decision)

자동연동을 **사용하지 않는다.** 사용자가 은행/카드사 웹에서 직접 내보낸(export) 거래내역 파일(CSV·엑셀)을 앱으로 **가져오기(import)** 한다.

## 근거

- 사용자의 금융 자격증명을 앱이 보관하지 않음 → 보안·신뢰 우위.
- 마이데이터/오픈뱅킹 허가 불필요 → 개인 개발로 공개 배포 가능.
- 규제 리스크 제거. 2인 사용 앱의 ROI 대비 자동연동 리스크가 과도.

## 결과 (Consequences)

- (+) 법적·보안 부담 최소화, 빠른 MVP.
- (−) 사용자가 수동으로 파일을 내려받는 마찰.
- (→) 핵심 난제가 "연동 안정성"이 아니라 **"이기종 export 파일 파싱·정규화"** 로 이동(인코딩 CP949/EUC-KR, 기관별 양식, 중복 제거). SMS/알림 파싱은 v2 후보, 자동연동은 영구 비목표.
