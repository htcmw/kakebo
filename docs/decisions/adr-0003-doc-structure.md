# ADR-0003: 문서 구조 — phase 정본 + views 인덱스

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted |
| 날짜 | 2026-06-18 |
| 단계 | Inception (meta) |

## 맥락 (Context)

문서를 단계(phase) 기준으로 묶을지, 범주(type) 기준으로 묶을지 논의했다.

- Phase 기반: 빌드 흐름·누락 방지에 강하나, 살아있는 문서가 단계 라벨과 어긋날 수 있음.
- Type 기반: 레퍼런스·코드 동형성에 강하나 범주 경계가 모호.
- "둘 다 관리"는 매력적이나 같은 내용을 두 곳에 복제하면 **불일치(드리프트)** 가 구조적으로 발생.

## 결정 (Decision)

**정본(문서 실물)은 phase 폴더에 한 벌만** 둔다. 범주(type) 보기는 `views/`의 **인덱스(링크 목록)** 로 제공하고, 내용은 절대 복제하지 않는다. ADR은 별도 시간순 로그로 유지한다.

## 근거

- 단일 정본 → 동기화할 "내용"이 없으므로 불일치 불가능.
- 링크 부패는 발생 가능하나 내용 불일치와 달리 기계적 검증 가능(CI 링크 체크).

## 결과 (Consequences)

- (+) "두 보기 분리 관리"를 불일치 위험 없이 달성.
- (−) views/인덱스 링크는 문서 이동·개명 시 갱신 필요.
- (→) 이 원칙은 ADR 인덱스(`decisions/README.md`) 등 모든 인덱스/뷰에 적용 — 뷰는 링크만, 내용은 정본에.
