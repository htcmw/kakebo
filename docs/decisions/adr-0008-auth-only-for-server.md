# ADR-0008: 인증은 서버(백업/동기화) 사용 시에만

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted |
| 날짜 | 2026-07-08 |
| 단계 | Construction |
| 갱신 | ADR-0002(매직링크 P0)를 부분 대체 |

## 맥락 (Context)

ADR-0007로 데이터 정본이 로컬이 되면서, 로컬 사용에는 로그인 이유가 사라진다. 인증은 S3 백업/복원처럼 **서버 기능을 쓸 때만** 필요하다. 사용자도 "인증은 서버 쓸 때만 필요, 그 전까지는 불필요"라고 명시했다.

기존 ADR-0002는 이메일 매직링크를 MVP 필수(P0)로 두었는데, 로컬 우선 전환으로 그 전제가 바뀐다.

## 결정 (Decision)

**로컬 MVP는 인증 없음.** 앱을 열면 바로 사용한다.

인증은 **서버 기능(백업/복원, 향후 공유·동기화)을 켤 때만** 요구한다. 그 인증 수단(매직링크 또는 클라우드 계정)은 Phase 2 서버 설계에서 확정한다.

→ **ADR-0002 갱신**: 매직링크 인증은 MVP P0가 아니라 **Phase 2(서버) 기능**으로 이동. FR-AU-01/02의 우선순위는 로컬 MVP 기준 P2/보류로 재조정.

## 근거

- 관문(로그인)을 핵심 로컬 경험에서 제거 → 마찰 0, 즉시 사용.
- 서버 없이 동작하므로 인증할 대상 자체가 없음.
- import 파이프라인 등 진짜 핵심에 집중(ADR-0002 근거의 연장).

## 결과 (Consequences)

- (+) 로컬 앱 진입 마찰 없음, 프라이버시(계정 수집 없음).
- (→) 서버 백업 시 무엇으로 인증할지(사용자 클라우드 계정, 앱 계정, S3 자격 등)는 아키텍처 Phase 2에서 결정.
- (변경) 목업의 로그인 화면은 Phase 2용으로 재배치. 로컬 MVP 진입 흐름은 로그인 없이 시작.
- (변경) ADR-0002 상태: **부분 대체(Superseded in part)** — 인증은 서버 단계로.
