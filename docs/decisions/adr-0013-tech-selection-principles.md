# ADR-0013: 기술 선택 원칙 (오픈소스·무료·활성 커뮤니티·과금유도 없음)

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted |
| 날짜 | 2026-07-08 |
| 단계 | Construction (meta) |

## 맥락 (Context)

사이드 프로젝트(부부 2인)로서 지속가능성과 데이터 소유권이 중요하다. 특정 벤더의 유료 전환 압박(과금 유도)이나 유지보수가 끊긴 도구에 묶이면 리스크가 크다.

## 결정 (Decision)

기술을 고를 때 다음 기준을 **모두** 만족하는 것을 우선한다.

1. **오픈소스 또는 무료** (가능하면 관대한 라이선스: MIT/Apache/BSD/PublicDomain).
2. **활성 커뮤니티** (최근 릴리스·이슈 응답·채택 규모).
3. **높은 완성도·성숙도** (프로덕션 사용 사례, 안정 버전).
4. **과금 유도 없음** — 무료 시작이 나중에 강제 유료로 전환되거나, 핵심 기능이 페이월 뒤에 있거나, 이탈 비용(egress·락인)이 큰 구조를 피한다.

유료·프로프라이어터리 서비스는 **선택적 편의**로만 쓰고, 항상 **무료/오픈 대체 경로**를 열어둔다(표준 인터페이스로 락인 방지).

## 현재 스택 감사 (2026-07)

| 기술 | 라이선스 | 무료 | 커뮤니티/성숙 | 과금유도 | 판정·대응 |
| :-- | :-- | :-- | :-- | :-- | :-- |
| React Native | MIT | ✅ | 매우 높음 | 없음 | ✅ |
| Expo SDK/CLI/Router | MIT | ✅ (영구) | 매우 높음 | 없음 | ✅ |
| **EAS**(Build/Update/Submit) | 서비스(프리티어) | 부분 | 높음 | ⚠ 있음 | **선택적** — 로컬 빌드/GitHub Actions·자가호스팅 업데이트로 우회(ADR-0011) |
| TypeScript | Apache-2.0 | ✅ | 매우 높음 | 없음 | ✅ |
| NativeWind | MIT | ✅ | 높음 | 없음 | ✅ |
| Gluestack UI v3(선택) | MIT | ✅ | 중상 | 없음 | ✅ |
| SQLite / expo-sqlite | PublicDomain / MIT | ✅ | 매우 높음 | 없음 | ✅ |
| Zustand | MIT | ✅ | 높음 | 없음 | ✅ |
| **백엔드 저장소**(Phase 2) | — | — | — | ⚠ 있음 | S3-호환 + **무료/오픈 우선**: Cloudflare R2(무료 10GB·egress $0) 기본, Backblaze B2 대안, 자가호스팅은 Garage/SeaweedFS. **MinIO 커뮤니티는 2026-02 아카이브라 제외**. (ADR-0012) |

결론: 클라이언트 스택은 전부 오픈소스·무료로 원칙 충족. 과금유도 소지는 **EAS**와 **백엔드 저장소** 두 곳뿐이며, 각각 우회·대체 경로를 표준화해 원칙을 지킨다.

## 결과 (Consequences)

- (+) 벤더 락인·유료 전환 압박 최소화, 지속가능.
- (−) 편의 서비스(EAS 클라우드 빌드 등)를 안 쓰면 셋업 손이 더 감 → 필요 시 프리티어 한도 내에서만.
- (→) 신규 기술 도입 시 이 4기준으로 심사하고, 과금유도 소지는 대체 경로를 함께 문서화한다.
- (→) ADR-0011(EAS 선택적), ADR-0012(백엔드 오픈 대안)에 반영.
