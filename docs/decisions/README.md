# ADR 인덱스 — 결정 지도

ADR(결정 기록)을 **카테고리**로 묶고, 각 결정이 *무엇을(how) · 왜(why) · 어디에/누가(where/who) · 상태 · 관계* 로 보이게 한 뷰입니다. 정본은 각 `adr-XXXX-*.md` 파일이며, 이 문서는 인덱스(링크)입니다.

## 한눈에 (관계도)

```mermaid
graph TD
  subgraph 제품·데이터수집
    A1[0001 자동연동 미사용<br/>export→import]
  end
  subgraph 금융로직[금융 로직 · 가계부 핵심]
    F4[0004 이용=지출<br/>정산=이체]
    F5[0005 내부 vs 외부 이체]
    F6[0006 할부 두 관점]
  end
  subgraph 아키텍처·인증
    R7[0007 로컬 우선<br/>+S3 백업]
    R8[0008 인증은 서버 쓸 때만]
    R2[0002 매직링크 인증]
  end
  subgraph 기술스택[기술 스택]
    T13[0013 기술선택 원칙]
    T10[0010 UI 스택]
    T11[0011 프론트엔드]
    T12[0012 백엔드]
  end
  subgraph 개발방식·문서[개발 방식 · 문서 메타]
    M3[0003 문서 구조]
    M9[0009 에이전틱 워크플로]
  end

  A1 --> F4
  F4 --> F5 --> F6
  R7 --> R8
  R7 --> T11
  R7 --> T12
  R8 -. 부분대체 .-> R2
  T13 ==> T10
  T13 ==> T11
  T13 ==> T12
  M9 -. 규율 .-> T11
```

범례: `→` 파생/연관 · `==>` 상위 원칙이 규율 · `-. 대체/규율 .->` 관계.

## 관계 요약

- **대체**: 0008 이 0002 를 부분 대체(인증을 서버 단계로).
- **상위 규율**: 0013(기술 원칙)이 0010·0011·0012를 규율.
- **파생**: 0007(로컬 우선)이 0008·0011·0012로, 0004가 0005·0006로 이어짐.

## 새 ADR 추가 규칙

1. 다음 번호로 `adr-XXXX-kebab.md` 생성(형식: 상태·날짜·단계 / 맥락 / 결정 / 근거 / 결과 / 관련).
2. 기존 결정을 바꾸면 옛 ADR **상태를 갱신**(대체/부분대체)하고 관계를 명시.
3. **이 인덱스**의 해당 카테고리에 한 줄 추가하고, 필요 시 관계도 갱신.
