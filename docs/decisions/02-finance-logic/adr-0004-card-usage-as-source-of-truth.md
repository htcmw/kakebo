# ADR-0004: 카드 이용내역 = 지출의 단일 원천, 카드 정산은 이체

| 항목 | 내용 |
|------|------|
| 상태 | ✅ Accepted |
| 날짜 | 2026-07-06 |
| 단계 | Inception → Construction |

## 맥락 (Context)

앱은 계좌(은행)와 카드를 모두 다룬다. 이때 같은 돈이 두 곳에 나타난다.

- 카드로 결제하면 카드 **이용내역**에 개별 결제 건이 남는다.
- 그 결제들은 월 1회 **카드 정산(카드대금)** 으로 묶여 결제계좌에서 빠져나간다.

카드 이용내역 파일과 은행 명세서를 둘 다 import하면, 개별 결제 합계(이용)와 카드대금 인출(정산)이 **같은 돈을 두 번** 잡아 지출이 이중집계된다. (예: 6월 카드 이용 합계 611,000원 + 7월 카드대금 인출 611,000원 = 1,222,000원으로 과대집계)

## 결정 (Decision)

**지출은 카드 이용내역(개별 결제)과 계좌의 직접 지출에서만 집계한다.** 카드 정산(카드대금 인출)은 지출이 아니라 계좌→카드사 간 **이체(내부 이동)** 로 분류해 수입·지출 집계에서 제외한다.

정산은 이용 시점에 이미 집계한 지출을 나중에 갚는 것이므로, 별도 지출로 세면 이중집계다.

## 근거

- 지출의 "발생 시점"은 결제(이용) 시점이지 정산 시점이 아니다.
- 이용내역은 가맹점·카테고리 정보를 담아 분류에 유리하다. 정산 인출 한 줄은 분류 정보가 없다.
- 원천을 하나로 고정하면 중복 판정(ADR/요구사항의 중복 키: 날짜+금액+적요+출처)과 이체 제외만으로 이중집계가 구조적으로 방지된다.

## 결과 (Consequences)

- (+) 카드·은행 파일을 함께 올려도 이중집계 없음.
- (+) 카테고리·공동/개인 분류가 결제 단위로 정확.
- (−) 카드 이용내역을 올려야 그 달 카드 지출이 집계된다. **은행 명세서만 올린 경우**, 카드대금 인출을 예외적으로 지출로 취급해야 하며(카테고리 세분화 불가), 앱은 "카드는 이용내역으로 가져오기"를 기본 가이드로 안내한다.
- (→) 정산 건을 이체로 인식하는 자동/수동 수단이 필요하다 → ADR-0005, FR-AC-04.
- (→) 할부가 섞인 정산도 통째로 제외되므로 할부는 이용내역 기준으로만 다룬다 → ADR-0006.
